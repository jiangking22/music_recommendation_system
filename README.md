# 多源音乐推荐系统

这是一个用于课堂展示的本地音乐推荐系统。系统支持输入歌曲名作为参考，联网检索原唱歌手，并从 QQ 音乐、网易云音乐、Apple Music、YouTube Music、Spotify 等平台召回歌曲，生成可解释的推荐列表。

当前版本重点展示三个能力：

- 多平台检索：兼容国际平台和国内平台，提升不同网络环境下的可用性。
- 原唱歌手识别：按输入歌曲语言选择候选来源；中文歌曲优先使用 QQ 音乐、网易云音乐，英文歌曲使用 Apple Music。系统不自动确认歌手，必须由用户在候选弹窗中选择。
- 分栏推荐控制：推荐结果按原唱歌手和其他歌手分为两栏，原唱歌手歌曲目标至少 3 首，其他歌手歌曲目标至少 5 首，兼顾相关性和丰富性。

## 运行方式

```powershell
cd C:\Users\KING\Desktop\信息内容安全\music_recommendation_system
python .\server.py
```

启动后打开终端显示的地址，默认是：

```text
http://127.0.0.1:8010/
```

项目不依赖第三方 Python 包。`server.py` 同时提供静态页面服务和国内音乐平台的本地代理接口。

## 使用方式

1. 在左侧输入框输入歌曲名，例如 `晴天`、`Love Story`、`七里香`。
2. 选择推荐语言，例如不限、华语、英语、日语、韩语、纯音乐。
3. 点击“按类型推荐 10 首”按钮后，系统才开始联网检索和生成推荐。
4. 如果系统发现多个可能原唱歌手，会弹窗让用户选择；如果没有可靠候选，也可以手动填写原唱歌手。
5. 推荐结果会展示歌曲名、歌手、语言、类型、来源平台、平台热度依据、歌手权重、推荐理由、在线查看链接和试听链接。

输入歌曲名不会自动推荐，必须点击推荐按钮后才开始检索，避免输入过程中频繁请求网络。

## 已完成功能

- 参考歌曲输入：支持输入一首或多首歌曲名，按换行、逗号或分号分隔。
- 语言选择：支持不限、华语、英语、日语、韩语、纯音乐等推荐方向。
- 多源检索：接入 QQ 音乐、网易云音乐、Apple Music、YouTube Music、Spotify。
- 本地代理：通过本地 `/api/search/domestic` 代理 QQ 音乐和网易云音乐，通过 `/api/search/streaming` 代理 YouTube Music 和 Spotify 搜索。
- 平台优先级自适应：中文输入优先 QQ 音乐、网易云音乐；英文输入优先 Apple Music、YouTube Music、Spotify；结果不足时再启用其他指定平台。
- 权威原唱候选：中文歌曲使用 QQ 音乐、网易云音乐检索候选，英文歌曲使用 Apple Music 检索候选。
- 歌手选择弹窗：系统不再自行确认原唱歌手，所有候选都由用户选择；没有候选时可手动填写。
- 歌手分栏控制：目标让原唱歌手歌曲至少 3 首，其他歌手歌曲至少 5 首，并分别按平台热度排序。
- 歌曲丰富性补充：原唱歌手作品会按语言优先级从指定平台共同召回。
- 推荐解释：每首歌曲展示推荐来源、歌手权重、类型、语言和推荐理由。
- 本地缓存：联网结果会写入浏览器本地缓存，后续检索可复用部分候选歌曲。

## 推荐流程

1. 用户输入参考歌曲并点击推荐。
2. 系统先判断参考歌曲语言：中文歌曲优先尝试 QQ 音乐、网易云音乐；英文歌曲优先尝试 Apple Music、YouTube Music、Spotify；数量不足时再启用其他指定平台补充。
3. 系统检索参考歌曲的可能原唱歌手。
4. 系统把检索到的可能歌手放入弹窗，不论分数高低都不自动确认，由用户选择最合适的原唱歌手；如果没有可靠候选，则手动填写。
5. 根据确认的原唱歌手召回该歌手热门歌曲，并召回同语言、同类型、同标签的相似歌曲。
6. 对候选歌曲进行去重、语言过滤、热度过滤和加权评分。
7. 通过“原唱歌手 / 其他歌手”分栏策略进行结果重排，输出最多 10 首推荐歌曲。

## 核心评分因素

- 歌曲热度：优先使用平台播放数、收藏数等指标；没有公开指标时使用平台搜索排名、榜单位置和接口返回热度。
- 语言匹配：是否符合用户选择的推荐语言。
- 类型匹配：歌曲曲风是否符合参考歌曲画像。
- 标签匹配：是否命中流行、民谣、电子、情歌、高能、学习、原声等标签。
- 歌手权重：如果歌曲来自已确认原唱歌手，会获得更高权重。
- 多样性重排：避免推荐列表过度集中在单一来源或单一风格。

## 平台说明

- QQ 音乐：中文歌曲的首选检索来源之一，通过本地服务代理搜索。
- 网易云音乐：中文歌曲的首选检索来源之一，通过本地服务代理搜索。
- Apple Music：英文歌曲的首选检索来源，也用于其他语言歌曲补充。
- YouTube Music：英文歌曲的优先补充来源，通过本地服务代理搜索。
- Spotify：英文歌曲的优先补充来源，通过本地服务代理搜索；可通过 `SPOTIFY_CLIENT_ID` 和 `SPOTIFY_CLIENT_SECRET` 使用官方授权。

如果某个平台请求慢、失败或返回结果少，系统会降低它在后续检索批次中的优先级。

## 课堂展示建议

1. 输入 `晴天`，点击推荐，展示系统如何识别周杰伦及其相关歌曲。
2. 输入 `Love Story`，展示英文歌曲下 Apple Music、YouTube Music、Spotify 的优先检索效果。
3. 断开或限制国际网络时，展示网易云和 QQ 音乐仍可返回国内歌曲。
4. 输入可能有多个歌手版本的歌曲，展示歌手选择弹窗。
5. 观察推荐卡片中的来源平台、平台热度依据和歌手权重，说明原唱歌手 / 其他歌手分栏策略。

- `app.js`：前端交互、多平台检索、原唱识别、推荐评分和结果渲染。
- `server.py`：本地静态服务，同时代理 QQ 音乐、网易云音乐、YouTube Music 和 Spotify 搜索接口。
- `课堂报告提纲.md`：课堂汇报可使用的结构化讲稿提纲。
